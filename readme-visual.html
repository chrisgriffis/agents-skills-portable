Strategy Framework

A portable architecture for persistent strategic awareness. Three-layer document model + AI agent infrastructure + state hygiene protocols. Plug in your specifics; the framework does the rest.

What This Is

Not a template collection. Not a prompt library. This is a complete information architecture for managing multi-actor strategic situations across months or years, using an AI agent as a persistent memory and analytical counterpart.

You supply: your actors, your terrain, your arcs, your goals. The framework supplies: the container structure, the merge discipline, the state hygiene protocols, the defect filters, the visual layer, and the agent behavior definitions that keep the whole thing honest across sessions.

The problem it solves

No single document can serve as both "what is my 3–5 year strategic position" and "what did Actor X say on Tuesday." Different time horizons and update frequencies demand different containers—connected by cross-references and a strict merge direction. Without this structure, strategic context degrades into a pile of notes that cannot predict anything.

Three-Layer Architecture

Information is stored in layers with different time horizons, update cadences, and audiences. Lower layers feed upper layers. Never the reverse.

Layer 1 — Career Arc Integrator

Long-Horizon Strategy Document

Structured sections. Kind and unkind reads. Evolution tracking with dated entries showing how understanding changes. 3–5 year horizon. Updated after strategic shifts or campaign postmortems. Contains: objective, self-assessment, cast, terrain, prior art, technical substance, active arcs.

Running People & Relationship Memory

Layer 3 — Campaign Tactical Dossier

Push-Specific Operational State

What you owe, to whom, by when. Deliverable tracking, logistics, ship targets. References Layer 2 entries by person-ID rather than duplicating them. Created per campaign. Frozen after event. Postmortem folds into Layer 1.

↑ Merge direction: always upward

Tactical summarizes into lifetime. Lifetime distills into career arc. Never duplicate downward. Contradictions between early and late reads are kept with dates—the evolution IS the insight.

State Hygiene Protocols

The AI agent has no persistent memory except what it writes to disk. Sessions end; context windows compress; models hallucinate confidence about things they haven't verified. These protocols exist to detect and correct the specific failure modes of LLM-based persistent-state systems.

Allup verification

Seven-step integrity protocol. Verifies the entire state tree is self-consistent and current, then produces a calibrated confidence rating. Invoked on demand or whenever a significant correction surfaces.

StepWhat it doesDefect it catches
0. Ground rulesPsychologically primes executor: step 5 will catch skipped workCorner-cutting, lazy assumptions
1. Self-consistency passRead ALL state files with tool calls (not from memory)Cross-file contradictions, stale facts
2. File reference cross-checkEnumerate every file; confirm all pointers are validOrphans, phantom references
3. Cascade check & FIXDetect inconsistencies AND remediate before proceedingTemporal drift, path rot, downstream claims
4. Self-flag debriefReport what was caught, missed, changed (with specifics)Silent failures, undisclosed gaps
5. Hallucination challenge"Did I actually do steps 1–3 or am I about to fabricate a good score?"Manufactured confidence
6. Context decay rating1–5 score on two independent failure surfacesUndetected staleness, compaction artifacts

Dual-Surface Decay Model

The context decay rating measures two independent failure surfaces. A session can be healthy on one and degraded on the other.

📄

Surface A: Document Freshness

Are state files verified-current via tool calls this turn? Degrades with time since last read and unverified agent edits. Unverified files cap rating at 3/5.

🧠

Surface B: Thread-Context Fidelity

Has the LLM’s own context window degraded during this session? Long sessions trigger compaction (lossy summarization of earlier turns). Model may confidently fabricate details from compacted memory.

Why two surfaces matter

Previous versions conflated "I read the files recently" with "my overall context is reliable." But a model can have all files fresh (Surface A = 5/5) while its own recall of what happened at turn 3 is unreliable because those turns have been compacted (Surface B = 2/5). Scoring both independently prevents false confidence.

Harden session close

Terminal action of every working session. Ensures nothing is lost at the session boundary.

  • Run full allup (all 7 steps, no abbreviation)
  • Sync operational state to archival copy in strategy repo
  • Write latest-handoff.md (overwrite, never append) — the bootstrap for next session
  • Commit strategy repo (feature branch → --no-ff merge → push)
  • Produce thread-transfer block (paste-ready cold-start message for next session)

Hardenout session close + export

Harden (all steps above) plus portable export: sync meta, templates, agent definitions, and skill definitions to the portable repo. Machine-loss survivability.

Defect Dimensions

The protocols are designed to catch failures across seven orthogonal dimensions:

DimensionExample failureDetection mechanism
Temporal driftPERT chart says "TODAY" for yesterday’s dateAllup step 3 (cascade check)
Cross-file inconsistencyDossier says X reports to A; trajectory says BAllup step 1 (self-consistency pass)
Orphan/phantom referencesREADME points to a file that was moved or deletedAllup step 2 (file cross-check)
Hallucinated confidenceAgent reports "4/5 freshness" without having read filesAllup step 5 (hallucination challenge)
Thread-context decayAgent fabricates details from compacted early turnsAllup step 6 Surface B
Cascade-on-fix errorsCorrecting a name in one file invalidates three othersAllup step 3 (fix before proceeding)
Session-boundary amnesiaNew session has zero context from predecessorHarden steps 3+5 (handoff + transfer block)
Why this exists

LLMs in persistent-state configurations exhibit a specific failure mode: they become more confident as the session lengthens while simultaneously becoming less accurate (due to context compression). The allup protocol forces verification at exactly the moments the agent is most likely to be wrong—session boundaries and after corrections. The hallucination challenge step is adversarial to the agent’s own incentive to report success, which is the point.

Operational / Archival Split

Agent state exists in two copies. Operational (hot) is read/written during sessions—may be dirty, ahead of archival. Archival (cold) is committed to the strategy repo during harden. They are explicitly allowed to drift; operational wins on conflicts. This avoids commit-on-every-write noise while maintaining recovery capability.

Role / actual power: How they got here / what they're protecting: Failure mode(s): Current relationship to you: What they want that you could plausibly give them: What they fear your initiative might trigger: <-- most important field Who they actually listen to: Last meaningful interaction / signal: Notes:

Hard Rules

  • Write the unkind read. Always. Charitable framing is how the dossier stops predicting things.
  • Capture priors BEFORE new data arrives (lets you see what changed vs. confirmed).
  • Radioactive language stays in the dossier. Never leaks into org-facing comms.
  • The dossier predicts; it does not prescribe. When it stops predicting, update the model.
  • Evolution tracking: don't overwrite old reads. Date them and add new ones alongside.

Adopting the Framework

To use this for your own strategic work:

  • Clone this repo. It is your machine-loss insurance and your framework reference.
  • Create a local strategy repo (private, never shared). This holds your Layer 1 + exhibits.
  • Create an agent state folder (wherever your AI tooling expects it). This is Layer 2 hot copy.
  • Adapt the agent templates in strategy-workspace/templates/ to point at your state paths.
  • Feed the appropriate SKILL.md as instruction context to your AI sessions.
  • Populate your dossier with actors. The cast schema tells you what fields to fill.
  • Run allup periodically to keep state honest. Harden at every session close.

When to Use This

Appropriate

Multiple actors with non-aligned interests. Career-level stakes. Months-to-years horizon. Strategic picture too complex for one document. AI counterpart maintaining state across sessions.

Overkill

Simple project management. Technical work with no political dimension. Short-term tactical problems. Fewer than 3 relevant actors. Situations where the answer is obvious.
