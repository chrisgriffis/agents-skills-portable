Strategy

Multi-layered personal strategy workspace. Document curation + AI-assisted state management + structured decision tracking.

Three-Layer Architecture

Information architecture for different time horizons and update cadences. No single document can serve as both career strategy and daily task tracker.

Layer 1 — Career Arc Integrator

Long-Horizon Strategy Document

Structured sections with evolution tracking. Dated reads showing how understanding changes. 3–5 year horizon. Updated after strategic shifts or campaign postmortems.

↓
Layer 2 — Agent Lifetime Dossier

Running People & Relationship Memory

Current reads on actors. Relationship dynamics, behavioral patterns, operating preferences. Updated every session. Persists across all campaigns. Role-tenure horizon.

↓
Layer 3 — Campaign Tactical Dossier

Push-Specific Operational State

Deliverable tracking, logistics, ship targets, actor positioning for this specific campaign. References lifetime dossier by person-ID. Frozen after event concludes.

↑ Merge direction: always upward

Tactical summarizes into lifetime. Lifetime distills into career arc. Never duplicate downward. Contradictions between early and late reads are kept with dates—the evolution IS the insight.

State Hygiene Protocols

The AI agent operates across sessions with no persistent memory other than what is written to disk. This creates a specific failure surface: state drift, hallucinated confidence, and cascade errors. The protocols below detect and correct these defects systematically.

Allup verification

Seven-step protocol. Verifies the entire state tree is self-consistent and current, then produces a calibrated confidence rating.

StepPurposeDefect it catches
0. Ground rulesRead-before-proceed disciplineLazy shortcuts, prior-turn assumptions
1. Self-consistency passRead every state file with tool callsCross-file contradictions, stale facts
2. File reference cross-checkEnumerate all files, confirm referencesOrphans, phantom pointers
3. Cascade check & FIXDetect AND remediate inconsistenciesTemporal drift, path rot, content claims
4. Self-flag debriefReport what was caught/missed/changedSilent failures, undisclosed gaps
5. Hallucination challengeSelf-interrogation: did I actually verify?Fabricated confidence scores
6. Context decay rating1–5 on two surfaces (see below)Undetected staleness, compaction artifacts

Dual-surface decay model

The context decay rating measures two independent failure surfaces:

📄

Surface A: Document Freshness

Are state files verified-current via tool calls this turn? Degrades with time since last read and unverified edits by the agent.

🧠

Surface B: Thread-Context Fidelity

Has the LLM’s own context window degraded? Long sessions trigger compaction (lossy summarization of earlier turns). Model may fabricate details it believes it remembers.

A session can score 5/5 on Surface A (all files read fresh) while scoring 2/5 on Surface B (model recall unreliable due to compaction). Both must be assessed independently.

Harden session close

Terminal action of any working session. Ensures nothing is lost at the session boundary.

  • Run full allup (all 7 steps, no abbreviation)
  • Sync operational state to archival copy in strategy repo
  • Write latest-handoff.md (the bootstrap document for next session)
  • Commit strategy repo (feature branch → --no-ff merge → push)
  • Produce thread-transfer block (paste-ready cold-start message)

Hardenout session close + export

Harden plus portable export: sync skills, agent definitions, and templates to the GitHub-hosted portable repo. Machine-loss survivability.

Springclean system audit

Full-system reconciliation via the housekeeper mixin. Nine-phase deep clean: enumerate all files, ingest completely, entity audit (name hygiene), cross-file consistency check, arc coherence assessment, severity catalog, optional Greene read, structured report, user-gated remediation. Use every 1–2 weeks or after multi-day gaps. Produces a Global Reconciliation Report using a standardized template.

Why this exists

LLMs in persistent-state configurations become more confident as sessions lengthen while simultaneously becoming less accurate (context compression). The allup protocol forces verification at exactly the moments the agent is most likely to be wrong—session boundaries and after corrections. The hallucination challenge step is adversarial to the agent’s own incentive to report success, which is the point.
